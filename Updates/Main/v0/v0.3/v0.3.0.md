## Summary
- 將舊版 logger 管理器遷移至 demonkernel/tools/logger 並重構為模組化架構
- 建立 logs/<session>/log 與 logs/<session>/json 雙目錄分流機制
- 實作雙檔案輸出（.log + JSON Lines）
- 將壓縮流程由同步改為非同步 tar.gz 背景處理
- 對齊 winston/pino 設計理念並補齊舊版未完成功能
- 新增 level gate、child logger、redaction、serializer、session 管理與 transports 抽象層
- 提供 TypeScript wrapper 供 kernel 與 pluginsManager 使用
- 完成測試、文件與遷移說明整合

## Changes
### Added
- 新增模組化 logger 架構於 demonkernel/tools/logger
- 新增 levels、redaction、serializer、session、transports、compress 子模組
- 新增雙檔案輸出機制（.log 與 JSON Lines .json）
- 新增 logs/<session>/log 與 logs/<session>/json 目錄分流結構
- 新增非同步 tar.gz 壓縮機制
- 新增 level gate 與 isLevelEnabled API
- 新增 child logger（支援 bindings 繼承）
- 新增錯誤序列化與 meta 結構化處理
- 新增 TypeScript wrapper（src/core/logger/index.ts）
- 新增 logger 測試套件（涵蓋 flush、shutdown、compression、redaction 等）
- 新增完整 logger 文件套件（overview、api、integration、migration、operations）
- 新增 tar 依賴與 logs/ 忽略規則設定

### Changed
- 將舊版 logger.js 遷移並重構為模組化架構至 tools/logger
- 日誌輸出根目錄統一為 demonkernel/logs/
- 原本單一目錄輸出改為 logs/<session>/log 與 logs/<session>/json 分流結構
- 原本同步壓縮流程改為非同步 tar.gz 背景壓縮
- 原本僅文字輸出改為同時輸出 .log 與 JSON Lines .json
- pluginsManager 與 runtime 由 console.* 改為使用新 logger
- TypeScript wrapper 對外 API 統一為 createKernelLogger/flushKernelLogs/shutdownKernelLogger
- ManagerLogger 型別維持相容但改由新 logger adapter 實作
- 預設 level 與 console 輸出策略重新定義（console 僅 warn/error/fatal）
- 測試與文件同步更新以符合新目錄與輸出結構

### Fixed
- N/A

### Removed
- 移除 plugins 與 manager 中直接使用 console.* 的紀錄方式

## Impact
- 日誌系統架構標準化並對齊主流 logger 設計，提升可維護性與可擴充性
- 日誌輸出結構調整為 session 分流目錄，舊有 log 讀取腳本需同步更新
- 壓縮改為非同步後可避免 shutdown 阻塞並提升執行效率
- 雙檔輸出（.log + JSON Lines）提升除錯與機器分析能力
- plugins 與 kernel 全面改用新 logger，統一紀錄行為
- 預設 console 僅輸出高嚴重度等級，降低終端雜訊
- 增加 redaction 機制，提升敏感資訊保護能力
- 增加 child logger 與 bindings，強化結構化紀錄能力
- 新增 tar 依賴，部署環境需確保相依套件可正常安裝

## Tests
- 新增 tools/logger/__tests__/logger.test.js 覆蓋核心行為（雙檔輸出、JSON Lines、level gate、child logger、redaction、serializer、flush/shutdown、非同步壓縮）
- yarn test 通過（33 tests）
- yarn build 通過
- yarn lint 通過

## Risks & Rollback
- 日誌目錄結構變更可能影響既有分析或讀取腳本
- 非同步壓縮若於極端情況下失敗可能產生未壓縮殘留資料
- 新增 tar 依賴在部分部署環境可能產生安裝相容性問題
- plugins 與 runtime 全面切換至新 logger 若有未覆蓋情境可能出現紀錄缺漏
- redaction 規則若設定不當可能誤遮蔽必要資訊

## Notes
- 本次重構屬於架構級調整，未來所有核心模組應統一透過新 logger 介面進行紀錄
- JSON 採用 JSON Lines 格式以利後續接入 ELK/ClickHouse 等分析系統
- session-based 目錄設計為後續實作 log rotation 與 retention policy 預留擴充空間
- 非同步壓縮策略設計為可替換式實作，未來可改為 worker thread 或 queue-based 處理
- TypeScript wrapper 作為 kernel 與 plugins 的唯一入口，避免直接操作底層 logger
- 文件已同步更新並提供 migration 指南供後續維護參考
