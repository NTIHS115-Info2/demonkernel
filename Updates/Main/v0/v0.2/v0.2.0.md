## Impact
- 所有插件必須改用 startupWeight 並移除 priority
- 所有 lifecycle 必須改為 throw-only 實作
- 舊版 PluginsManager API 不再可用
- 依賴版本必須精確匹配
- 插件若存在循環依賴將無法啟動
- 應用層啟動流程必須改為透過 PluginsManager 管理
- 舊 LLM 專屬管理接口將完全失效

- 新增虛假插件動態建立測試框架（skill/system、manifest、entry、dependencies）

- 新增 index integration 測試驗證 src/index.ts 已走 manager 啟動路徑

## Risks & Rollback
- 屬於破壞性升級，舊插件若未改為 startupWeight 與 throw-only 將無法啟動
- 依賴版本必須精確匹配，若 manifest 未同步更新將導致 blocked 或失敗
- 循環依賴將直接阻斷啟動流程
- 舊 LLM 專屬 API 已移除，若外部仍呼叫將立即失效
- Rollback 需回退至舊版 PluginsManager 並恢復 priority 與 Result 型契約
- Rollback 同時需還原 src/index.ts 啟動流程與 SDK 型別

## Notes
- 本次為架構級重構，重點在建立可擴展的依賴感知插件生命週期模型
- 未提供舊版相容層，建議所有插件同步完成契約升級後再部署
- 依賴等待未設 timeout，設計上採取強一致模型
- 版本匹配採精確字串比對，不支援 semver range
- 建議後續新增插件時避免循環依賴設計
- 建議插件內部避免自行捕捉並吞噬錯誤，應交由 manager 統一處理
- 本次為後續插件市場化與模組化擴展打基礎
