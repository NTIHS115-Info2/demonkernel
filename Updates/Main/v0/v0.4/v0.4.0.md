## Impact
- pluginsManager 不再將 cycle 一律視為阻塞，改以 SCC 與 startupWeight 規則決定 onlineMany 啟動順序
- 多依賴互依場景的啟動結果改為可預期且 deterministic
- 依賴分析報告由 blocked cycle 轉為保留 report.cycles 供後續判讀
- 相關測試案例、遷移文件、總覽文件與 manifest schema 說明已同步更新，降低後續擴充與維護成本

## Tests
- 更新 pluginsManager 測試案例以符合新的循環依賴策略
- 新增多依賴互依圖測試情境以驗證 SCC 分析與 startupWeight 排序邏輯
- 移除舊有「cycle blocked」預期並改為檢查 report.cycles 與 deterministic 啟動順序
- 

## Risks & Rollback
- 循環依賴不再被阻擋後，若 plugin 在實作上假設依賴已完全 online，可能在 SCC 內啟動順序下出現初始化競態或部分功能未就緒的情況
- 舊 plugin 若依賴「cycle 會被 blocked」的行為，可能出現預期外啟動
- 權重設定不當可能導致啟動順序不符合設計意圖

## Notes
- 本次變更僅調整 pluginsManager 的依賴分析與啟動排序策略，未改動 plugin runtime 介面
- cycle 資訊仍會保留於 report.cycles 供除錯與診斷使用
- plugin 若存在互依關係，建議透過 startupWeight 或延遲初始化機制確保功能可正常啟動
- 相關文件與 manifest schema 已同步更新
