## Changes
### Added
- 新增 system plugin llm-remote-gateway 用於提供 OpenAI 相容的 remote LLM API 呼叫能力
- 新增 capability system.llm.remote.chat.stream、system.llm.remote.models.list、system.llm.remote.health.check
- 新增 remote strategy 模組結構（payload、SSE stream、messageValidator、error handling、constants、types）

### Changed
- 將舊版 llamaServer 的 remote LLMStream 實作遷移並重構為符合新插件架構的 llm-remote-gateway system plugin
- 將原本硬編程的 API 參數與配置改為由 options / online 輸入提供
- 將舊版 stream 呼叫方式對齊新的 capability send(action, input) 調用模式
- 重構 SSE stream 處理流程與 message validation 邏輯以符合 OpenAI 相容 API 格式
- 調整插件結構以符合 pluginsManager、capability schema、logger 與 manifest 規範
- 將原本單一實作拆分為策略與功能模組（payload、sse、validator、errors、types）以提升可維護性與測試性

- N/A

### Removed
- 移除舊版 llamaServer 中 local 與 server 模式相關能力在新插件中的支援
- 移除舊插件中硬編程的 API 參數與預設配置讀取方式
- 移除對本地模型與內建伺服器啟動邏輯的依賴
- 移除舊版 LLMStream 與舊插件結構對新系統架構不相容的對接方式

## Impact
- 舊版 llamaServer 的 remote LLMStream 功能已由新的 system plugin llm-remote-gateway 取代
- 所有遠端 LLM 呼叫需改為透過 system.llm.remote.* capability 進行
- 原本依賴硬編程配置或本地 / server 模式的使用方式將不再適用
- 新的插件僅提供 OpenAI 相容 API 的 remote 能力並完全遵循 pluginsManager、capability 與 logger 規範

- 新增 llm-remote-gateway.integration.test.ts 整合測試以驗證新 system plugin 與 pluginsManager、capability registration、對外調用流程的對接正確性

## Risks & Rollback
- 舊系統若仍依賴 llamaServer 的 local 或 server 功能將無法運作
- 舊版直接呼叫 LLMStream 的流程若未改為 capability 呼叫可能導致相容性問題
- remote API 行為依賴 OpenAI 相容介面，若第三方服務實作差異可能影響 stream 回傳
- 回滾方式：移除 llm-remote-gateway system plugin 並恢復舊版 llamaServer 插件即可恢復原本架構與調用流程

## Notes
- 本次遷移僅重現舊版 llamaServer 的 remote 能力並整合至新插件架構，未包含 local 與 server 模式
- 新插件所有參數皆改由 options / online 輸入提供以避免硬編程設定
- remote API 呼叫統一採用 OpenAI 相容格式以簡化未來多模型服務整合
- 插件內部採策略模組化設計（payload、SSE、validator、errors、types）以提升可維護性與測試性
