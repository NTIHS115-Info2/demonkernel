## Summary
- 新增 talk-engine system plugin，提供可重用對話引擎能力

## Changes
### Added
- 新增 talk.nostream 與 talk.stream action，並提供 capability alias system.talk.engine.nostream/stream
- 新增模組化 local strategy（input 正規化、stream collector、relay queue）
- 新增可選 Discord relay，預設啟用並整合 typing.start/typing.stop

### Changed
- send 契約改為 action-only 並統一經 capability registry 呼叫 llm/discord provider

### Fixed
- relay 失敗時改為固定錯誤回覆並保證 typing.stop finally 執行
- relay queue 採 FIFO，避免多訊息回覆交錯

### Removed
- N/A

## Impact
- 上層插件可直接透過 capability registry 調用對話能力，不需重寫 llm payload 組裝
- talk-engine 可作為 Discord 對話編排中樞，降低 I/O 與推理耦合

## Tests
- 新增 tests/systemPlugins/talk-engine.test.ts 覆蓋 lifecycle、payload 轉換、stream/no-stream、relay 與 FIFO
- 新增 tests/pluginsManager/talk-engine.integration.test.ts 驗證 capability discover/registry

## Risks & Rollback
- relay 依賴 system.discord.* 與 system.llm.remote.chat.stream provider，若缺失會在 online 階段失敗
- talk.stream 透傳下游 emitter，呼叫方需自行處理事件生命週期

## Notes
- manifest 依賴固定為 llm-remote-gateway@1.0.0 與 discord@0.2.0
- 第一版策略為單輪無歷史，talker 會轉成 <sender=...> 前綴
