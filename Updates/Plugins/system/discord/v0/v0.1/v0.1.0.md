## Summary
- 新增 Discord system plugin，提供 conversation.stream 與 message.send 的 I/O-only 能力遷移

## Changes
### Added
- 新增 local strategy：Discord 登入、訊息過濾、inbound stream 輸出
- 新增 action-only send 路由（conversation.stream / message.send）
- 新增 capability 宣告：system.discord.conversation.stream 與 system.discord.message.send
- 新增 README 文件與註解說明

### Changed
- send 介面採 action-only，不保留舊版 func 呼叫格式

### Fixed
- 非 owner DM 統一固定回覆文案，避免未授權私訊進入對話流

### Removed
- 不遷移 slash commands、TalkToDemon 與句子分段回覆能力

## Impact
- 上層可透過 capability registry 取得 Discord 輸入事件流與輸出發送能力
- owner DM 與 mention/reply 可被統一路由到外部流程

## Tests
- 新增 tests/systemPlugins/discord.test.ts 驗證 lifecycle、filter、send 與錯誤路徑
- 新增 tests/pluginsManager/discord.integration.test.ts 驗證 capability discover/registry 路徑
- yarn test 全量通過

## Risks & Rollback
- 若 Discord API 權限不足或憑證錯誤，online/send 會拋出錯誤
- 回滾方式：移除 systemPlugins/discord 與其 UpdateLog 後重新 build

## Notes
- 密鑰統一由 SecretsManager 讀取（DISCORD_TOKEN / DISCORD_USER_ID / DISCORD_CHANNEL_ID）
