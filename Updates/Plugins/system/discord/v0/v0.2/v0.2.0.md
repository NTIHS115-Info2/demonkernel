## Summary
- discord system plugin 新增 typing session 控制能力，供對話編排插件引用

## Changes
### Added
- 新增 typing.start / typing.stop action 及 capability alias system.discord.typing.start/stop
- 新增 typingSessionManager 模組並實作 reference count + heartbeat sendTyping 機制

### Changed
- plugin.manifest.json 升級至 0.2.0，io/capabilities 文件同步納入 typing 契約
- online options 新增 typingIntervalMs 設定欄位

### Fixed
- offline 階段會清理全部 typing session，避免殘留 timer
- typing.stop 會在 refCount 歸零時正確停止該 channel session

### Removed
- N/A

## Impact
- 其他系統插件（例如 talk-engine）可直接透過 capability registry 控制 Discord typing 狀態
- Discord I/O 插件從單純收發訊息升級為可提供即時輸入狀態能力

## Tests
- 擴充 tests/systemPlugins/discord.test.ts 驗證 typing start/stop、reference count、heartbeat 與 offline 清理
- 擴充 tests/pluginsManager/discord.integration.test.ts 驗證 typing capabilities discover/registry

## Risks & Rollback
- typing 心跳依賴 channel 支援 sendTyping，若頻道型別或權限不符會產生日誌警告
- 錯誤設定過短 typingIntervalMs 可能增加 API 請求頻率

## Notes
- typing action 未提供自動超時關閉，需由呼叫端顯式 stop 或 offline 清理
- 現有 conversation.stream/message.send 契約保持相容
